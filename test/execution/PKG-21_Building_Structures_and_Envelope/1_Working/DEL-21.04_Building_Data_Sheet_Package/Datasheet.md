## Conditions

**Deliverable Purpose:**
Defines and substantiates building in accordance with ER requirements.
**Source:** Decomposition line 516

**Anticipated Data Sheet Artifacts:**
1. **Building system data sheet** — Overall building parameters, systems summary
2. **Door/window schedules** — Complete schedules of all doors and windows

**Building System Data Sheet Content:**
- Building identification (MCC building, operator shelters)
- Dimensions (footprint, height)
- Structural system (framing type, materials)
- Envelope system (cladding, roofing, insulation)
- Foundation type
- Design loads (snow, wind, seismic)
- Occupancy classification
- Fire rating (if applicable)

**Door Schedule Content:**
- Door mark/number
- Door type (personnel, overhead, fire-rated)
- Size (width × height)
- Material (steel, aluminum, etc.)
- Fire rating (if applicable)
- Hardware (hinges, lockset, closer, panic hardware)
- Remarks/notes

**Window Schedule Content:**
- Window mark/number
- Window type (fixed, operable)
- Size (width × height)
- Glazing type (single, double, Low-E)
- Frame material
- Remarks/notes

**Source:** Typical building data sheet and schedule content

## Construction

**Data Sheet Organization:**

**Building System Data Sheet:**
- General information (project, building name, location)
- Building parameters (dimensions, areas, occupancy)
- Structural system summary
- Envelope system summary
- Building services summary (if applicable)
- Design criteria summary (loads, codes)
- Performance criteria (thermal, fire, accessibility)

**Door Schedule:**
- Tabular format with columns: Mark, Type, Size, Material, Fire Rating, Hardware, Remarks
- Coordinate with door specification (DEL-21.02) and drawings (DEL-21.01)

**Window Schedule:**
- Tabular format with columns: Mark, Type, Size, Glazing, Frame, Remarks
- Coordinate with window specification (DEL-21.02) and drawings (DEL-21.01)

**Source:** Standard building data sheet and schedule formats
