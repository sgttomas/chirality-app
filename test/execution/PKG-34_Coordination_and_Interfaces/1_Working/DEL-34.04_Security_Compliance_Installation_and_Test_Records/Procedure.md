# Procedure: DEL-34.04 Security Compliance Installation & Test Records

## Purpose

Defines process for maintaining security compliance and producing **Security Compliance Installation & Test Records** within **PKG-34 Coordination & Interfaces**.

**Objective:** Maintain CBSA and Transport Canada security compliance during construction at secure port facility (supports OBJ-5 Terminal Continuity).

**Deliverable type:** Record | **Responsible:** D&B Contractor (QA/QC)

## Prerequisites

**Dependencies:** See `_DEPENDENCIES.md` (NOT_TRACKED)
**Upstream inputs:** Project security plan — **TBD**; Facility Security Plan — **TBD**; Personnel roster — **TBD**; CBSA/TC clearance portal access — **TBD**

**Reference materials:** Facility Security Plan (Fraser Surrey Terminal, approved by Transport Canada); CBSA border security regulations; Transport Canada MTSR; Employer's Requirements — **TBD**

**Personnel:** Security coordinator (D&B Contractor); Project manager; Site access control personnel; All construction personnel (security briefing)

## Steps

### Step 1: Security Planning and Setup (Before construction mobilization)

**1.1 Security Coordination Setup**
- Designate D&B Contractor security coordinator
- Establish liaison with Fraser Surrey Terminal security management
- Review Facility Security Plan requirements
- Coordinate with CBSA and Transport Canada (introduce project, confirm clearance process)
- **Verification:** Security coordination established

**1.2 Identify Personnel Requiring Clearances**
- Compile personnel roster (all construction personnel requiring restricted area access)
- Identify clearance level required per Facility Security Plan and work scope
- Prioritize critical personnel (early mobilization)
- **Verification:** Personnel roster compiled

### Step 2: Obtain Personnel Security Clearances (6-8 weeks before required access)

**2.1 Submit Clearance Applications**
- Collect personnel information (full name, DOB, citizenship, work history, contact)
- Complete clearance application forms (CBSA/TC portal or designated forms)
- Submit applications with adequate lead time (minimum 6 weeks) — **TBD**
- Track submission confirmations
- **Verification:** Applications submitted; confirmations received

**2.2 Track Clearance Status**
- Monitor clearance processing status (weekly follow-up)
- Respond promptly to CBSA/TC requests for additional information
- Escalate delays affecting critical personnel mobilization
- **Verification:** Clearance status tracked; delays escalated

**2.3 Receive and Record Clearance Approvals**
- Receive clearance approvals (CBSA/TC authorization letters or certificates)
- Record approvals in personnel clearance register (name, clearance number, issue date, expiry date)
- File clearance documents securely (confidential)
- **Verification:** Clearances recorded; documents filed securely

### Step 3: Implement Site Access Control (Construction phase)

**3.1 Issue Site Access Badges**
- Issue badges only to personnel with valid security clearances
- Badge format per Guidance.md Example 2 (name, company, badge number, valid dates, clearance level, photo)
- Maintain badge issuance log (badge number, personnel name, issue date, return date)
- **Verification:** Badges issued to cleared personnel only

**3.2 Manage Visitors**
- Visitor check-in at site entrance (name, company, purpose, host)
- Issue visitor badges (temporary, distinctive from personnel badges)
- Assign escort (visitor must be escorted at all times in restricted areas)
- Log visitor time in/out
- **Verification:** Visitors logged and escorted

**3.3 Control Vehicle Access**
- Issue vehicle access permits to authorized vehicles
- Vehicle inspection at entry (as required by security level or threat assessment)
- Log vehicle entry/exit
- **Verification:** Vehicle access controlled

**3.4 Monitor Restricted Area Access**
- Log restricted area access (who, when, purpose, authorization)
- Verify authorization before granting access
- Conduct periodic access audits (random checks, log reviews)
- **Verification:** Restricted area access logged and authorized

### Step 4: Security Training and Awareness (Ongoing)

**4.1 Conduct Security Briefings**
- Security briefing for all personnel before site access (security protocols, MARSEC levels, incident reporting, emergency procedures)
- Refresher briefings as needed (MARSEC level changes, security incidents)
- Maintain training records (attendee name, date, topics covered, trainer signature)
- **Verification:** All personnel briefed before site access

**4.2 MARSEC Level Awareness**
- Monitor current MARSEC level (Transport Canada notifications, Terminal security)
- Communicate MARSEC level changes to all personnel
- Implement enhanced security measures per MARSEC level requirements
- **Verification:** Personnel aware of current MARSEC level; enhanced measures implemented

### Step 5: Security Incident Management (As needed)

**5.1 Report Security Incident**
- Immediately report security incidents to:
  - D&B Contractor security coordinator and project management
  - Fraser Surrey Terminal security
  - CBSA, Transport Canada (as required by incident type and severity)
  - Law enforcement (if criminal activity suspected)
- Secure incident scene
- **Verification:** Incident reported to appropriate parties

**5.2 Investigate Security Incident**
- Conduct investigation (per Guidance.md Example 3):
  - Document incident details (date, time, location, description, personnel involved)
  - Interview witnesses
  - Identify contributing factors
  - Document immediate actions taken
  - Identify corrective actions to prevent recurrence
- Prepare security incident report
- **Verification:** Investigation conducted; report prepared

**5.3 Implement Corrective Actions**
- Implement corrective actions per investigation
- Track corrective actions to closure
- Update security procedures or protocols if needed
- Communicate lessons learned
- **Verification:** Corrective actions implemented

### Step 6: Regulatory Compliance and Inspections (Ongoing and as requested)

**6.1 Maintain Compliance Records**
- Maintain personnel clearance register (current and accurate)
- Maintain site access logs (badge issuance, visitors, vehicles, restricted areas)
- Maintain security training records
- Maintain security incident reports
- **Verification:** Compliance records current

**6.2 Prepare for CBSA/TC Inspections**
- Respond promptly to inspection requests
- Prepare records for inspector review (clearance register, access logs, training records, incident reports)
- Facilitate inspector site access and records review
- **Verification:** Inspection facilitated; records available

**6.3 Address Inspection Findings**
- Review inspection findings and non-conformances (if any)
- Prepare corrective action plan
- Implement corrective actions within required timeframe
- Submit corrective action completion evidence to CBSA/TC
- **Verification:** Non-conformances addressed; corrective actions completed

### Step 7: Compile and Archive Security Compliance Records (Ongoing and at project closeout)

**7.1 Compile Security Compliance Records Package**
- Personnel clearance records (roster, applications, approvals)
- Site access control records (badge issuance, visitor logs, vehicle logs, restricted area logs)
- Security training records (briefing attendance, topics, trainers)
- Security incident reports and investigations
- Regulatory inspection reports and corrective actions
- CBSA/TC correspondence
- **Verification:** Records compiled

**7.2 Archive Records Securely**
- Archive in secure storage with controlled access per CBSA/TC requirements
- Electronic storage with encryption and access logging
- Retention per CBSA/TC requirements (typically 7+ years) — **TBD**
- Provide copy to Employer (as required) — **TBD**
- **Verification:** Records archived securely per retention requirements
