# Procedure: DEL-31.01 Record Design Drawing Set

## Purpose

This procedure defines the process for producing and managing **Record Design Drawing Set** within **PKG-31 Documentation & Deliverables**.

**Objective:**
To produce a complete, accurate, and compliant set of Record Design Drawings that document the final approved design intent for the Canola Oil Transload Facility Phase 1, in accordance with Employer's Requirements (ER) and VFPA standards.

**Source:** Decomposition DEL-31.01 (line 686); _CONTEXT.md

**Deliverable type:** Drawing Set
**Responsible party:** D&B Contractor

**Source:** _CONTEXT.md

**Scope:**
This procedure addresses the production of Record Design Drawings from design completion through final issuance. It applies to all disciplines contributing to the drawing set (Civil, Structural, Mechanical, Process, Electrical, I&C, Marine, Buildings, Fire Protection).

## Steps

### Step 1: Planning and Scoping

**Objective:** Establish the scope, schedule, and resource plan for Record Design Drawing production.

**Activities:**

1.1. **Develop Drawing Register:**
   - Compile a comprehensive list of all required drawings across all disciplines
   - Assign drawing numbers per project numbering system
   - Identify drawing titles, disciplines, and anticipated issue dates
   - **Source:** **ASSUMPTION** per document management practice; **TBD** — project drawing numbering system

1.2. **Coordinate with Discipline Leads:**
   - Confirm drawing scope with each discipline lead
   - Identify interface drawings and coordination requirements
   - **Source:** **ASSUMPTION** per multi-discipline coordination

1.3. **Establish Drawing Production Schedule:**
   - Align drawing production with overall project design schedule
   - Identify critical path drawings (e.g., permit-critical, early construction)
   - Allocate resources (designers, checkers) to drawing production tasks
   - **Source:** **ASSUMPTION** per project scheduling practice

1.4. **Obtain VFPA Standards and Templates:**
   - Acquire VFPA drawing standards and any required templates
   - Develop project-specific CAD templates compliant with VFPA requirements
   - **Source:** Specification FR-03; Guidance (VFPA Standards Compliance section)

1.5. **Establish Quality Procedures:**
   - Confirm drawing checking and approval procedures per project Quality Plan
   - Identify checker and approver roles and assignments
   - **Source:** Specification QR-01, QR-02; **ASSUMPTION** per QA/QC planning

**Deliverables from Step 1:**
- Draft Drawing Register
- Drawing Production Schedule
- CAD Templates and Standards Document
- Quality Assurance Plan for drawing production

**Verification:**
- Drawing Register reviewed and approved by Project Manager and discipline leads
- Schedule integrated with overall project schedule

### Step 2: Drawing Production

**Objective:** Produce draft drawings for each discipline based on finalized design calculations and specifications.

**Activities:**

2.1. **CAD Drafting:**
   - Designers/drafters produce drawings using project CAD templates
   - Incorporate design information from calculations, specifications, and design basis documents
   - Apply VFPA-compliant symbology, annotations, and formatting
   - **Source:** Specification PR-02; **ASSUMPTION** per CAD production practice

2.2. **Self-Check:**
   - Originator performs self-check for completeness, accuracy, and standards compliance
   - Correct errors and omissions identified during self-check
   - **Source:** **ASSUMPTION** per engineering QC practice

2.3. **Discipline Coordination:**
   - Participate in interdisciplinary coordination meetings
   - Identify and resolve clashes and interface issues (manual review or 3D clash detection if models used)
   - Update drawings to reflect coordination outcomes
   - **Source:** Specification FR-04; Guidance (Coordination and Clash Resolution section)

2.4. **Drawing Register Updates:**
   - Update drawing register with drawing status (e.g., "In Progress", "Ready for Check")
   - Track completion progress against schedule
   - **Source:** **ASSUMPTION** per project tracking practice

**Deliverables from Step 2:**
- Draft drawings (all disciplines)
- Updated Drawing Register

**Verification:**
- Originator sign-off on self-check completion
- Coordination meeting minutes documenting issue resolution

### Step 3: Technical Checking and Review

**Objective:** Conduct independent technical review of drawings to ensure accuracy, completeness, and standards compliance.

**Activities:**

3.1. **Independent Technical Check:**
   - Qualified checker reviews drawings for technical accuracy, completeness, and compliance with specifications and standards
   - Checker verifies calculations, design assumptions, and technical content against source documents
   - Checker records comments and issues in comment tracking system
   - **Source:** Specification QR-02; Verification method 1 (Technical Accuracy Review)

3.2. **Interdisciplinary Check:**
   - Drawings reviewed by adjacent disciplines to confirm interface coordination
   - Cross-discipline issues identified and recorded
   - **Source:** Specification FR-04; Verification method 4 (Multi-Discipline Coordination Review)

3.3. **VFPA Standards Compliance Check:**
   - Review drawings for compliance with VFPA standards (symbology, formats, annotations, scales)
   - Verify title blocks, coordinate systems, and other VFPA-specific requirements
   - **Source:** Specification FR-03, PR-02; Verification method 3 (Format and Standards Compliance)

3.4. **Completeness Check:**
   - Verify all drawings listed in Drawing Register are present
   - Confirm no missing sheets or incomplete information
   - **Source:** Verification method 2 (Completeness Check)

3.5. **Comment Resolution:**
   - Originator addresses all checker comments
   - Comments closed out when resolution confirmed by checker
   - Drawings revised as needed
   - **Source:** **ASSUMPTION** per QA/QC comment management practice

**Deliverables from Step 3:**
- Checked drawings with checker sign-off
- Comment tracking log showing all comments closed

**Verification:**
- Checker sign-off on drawings (signature/stamp on drawing or in tracking system)
- Comment tracking log reviewed and approved by Project Manager

### Step 4: Stakeholder Review

**Objective:** Obtain review and input from internal stakeholders, Employer (if required), and regulatory authorities (as applicable).

**Activities:**

4.1. **Internal Design Review:**
   - Drawings reviewed by D&B Contractor project team (Project Manager, discipline leads, construction manager, etc.)
   - Constructability, operability, and maintainability considerations evaluated
   - Internal review comments documented and addressed
   - **Source:** Verification method 5 (Stakeholder Review); Guidance (Constructability, Operability sections)

4.2. **Employer Review (if required):**
   - Submit drawings to Employer for review per contract requirements
   - **TBD** — Specific Employer review and approval requirements per Employer's Requirements and contract
   - Address Employer comments and resubmit as needed
   - **Source:** **ASSUMPTION** per typical design-build contract procedures

4.3. **Regulatory Review (as applicable):**
   - Submit drawings to regulatory authorities (VFPA, Transport Canada, municipal building departments, etc.) for permit review
   - Coordinate with PKG-32 (Regulatory & Permits) for submission timing and requirements
   - Address regulatory comments and resubmit as needed
   - Obtain regulatory approvals or confirmations as required
   - **Source:** Specification IR-03; cross-reference PKG-32

4.4. **Comment Incorporation:**
   - Incorporate approved comments from Employer and regulatory authorities
   - Update drawings and drawing register to reflect revisions
   - **Source:** **ASSUMPTION** per review cycle management

**Deliverables from Step 4:**
- Reviewed drawings with stakeholder comments addressed
- Employer approval or acceptance (if required)
- Regulatory approval letters or permits (as applicable)
- Updated Drawing Register reflecting latest revisions

**Verification:**
- All stakeholder comments closed out and documented
- Required approvals obtained and recorded

### Step 5: Final Approval and Issuance

**Objective:** Obtain final internal approval and issue Record Design Drawings for construction use.

**Activities:**

5.1. **Final QA Review:**
   - Final quality assurance review to confirm all checks completed, comments resolved, and approvals obtained
   - Verify drawing set completeness and consistency
   - **Source:** **ASSUMPTION** per quality management final checks

5.2. **Approval Sign-Off:**
   - Authorized approver (e.g., Chief Engineer, Project Manager) provides final approval sign-off
   - Approval recorded on drawing title blocks and in drawing register
   - **Source:** Specification QR-03

5.3. **Drawing Status Update:**
   - Update drawing status to "Approved for Construction" (or equivalent project-specific status)
   - Update drawing register with issue date and status
   - **Source:** **ASSUMPTION** per document status conventions

5.4. **Electronic and Hard Copy Preparation:**
   - Prepare electronic deliverables (CAD native files, PDFs) per Specification requirements
   - Prepare hard copy sets if required
   - **Source:** Specification (Documentation section)

5.5. **Drawing Transmittal:**
   - Prepare drawing transmittals documenting issued drawings
   - Transmit drawings to Employer, construction team, and other recipients per distribution matrix
   - Upload to project document management system
   - **Source:** **ASSUMPTION** per document transmittal procedures

5.6. **Filing and Archiving:**
   - Place issued drawing copies in `3_Issued/` folder
   - Archive superseded revisions (if any)
   - Maintain drawing register as master record of issued drawings
   - **Source:** README.md (How to use the framework); **ASSUMPTION** per document archiving practice

**Deliverables from Step 5:**
- Issued Record Design Drawing Set (electronic and hard copy)
- Drawing Transmittals
- Updated Drawing Register (final)
- Approvals and sign-offs documented

**Verification:**
- Approver sign-off obtained and recorded
- Transmittals sent and receipt confirmed (if applicable)
- Document management system updated with issued drawings

## Verification

**Verification activities for Drawing Set deliverables:**

The following verification methods are applied at appropriate stages throughout the procedure:

1. **Technical Accuracy Review** (Step 3.1)
   - Independent technical review by qualified checker
   - Verification of calculations, design assumptions, and technical content
   - **Acceptance Criterion:** Checker sign-off obtained; no unresolved technical issues

2. **Completeness Check** (Step 3.4)
   - Verification that all required drawings are included
   - Cross-check against drawing register and deliverable requirements
   - **Acceptance Criterion:** Drawing register 100% complete; no missing drawings

3. **Format and Standards Compliance** (Step 3.3)
   - Review for compliance with VFPA drawing standards
   - Check symbology, annotations, title blocks, scales, coordinate systems
   - **Acceptance Criterion:** Full compliance with VFPA standards confirmed

4. **Multi-Discipline Coordination Review** (Step 2.3, Step 3.2)
   - Clash detection and coordination verification
   - Interface point confirmation
   - **Acceptance Criterion:** No unresolved interdisciplinary conflicts

5. **Stakeholder Review** (Step 4)
   - Internal design review (D&B Contractor team)
   - Employer review and approval (as required per contract)
   - Regulatory authority review (as required for permitting)
   - **Acceptance Criterion:** All stakeholder comments addressed and closed; required approvals obtained

**Overall Acceptance Criteria:**
- All technical review comments resolved and closed
- Checker and approver sign-offs obtained on all drawings
- Drawings issued with "Approved for Construction" or equivalent status
- Compliance with VFPA standards confirmed
- Employer acceptance obtained (if required per contract)
- Drawing register complete and accurate

**Source:** Specification (Verification section); **ASSUMPTION** per project quality procedures
