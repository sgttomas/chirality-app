This procedure defines the process for **producing** the **Security System Design Drawing Set** within **PKG-24 Security Systems** for the Canola Oil Transload Facility.

**Deliverable objective:** Defines the design arrangement and details for security system per ER requirements.

**Responsible party:** D&B Contractor

**Procedure scope:** This procedure covers the workflow from initial design concept through final drawing issuance, including design development, drafting, interdisciplinary coordination, checking, and approval.

## Steps

### Step 1: Project Initiation and Requirements Review

**Objective:** Understand security system requirements and project context.

**Activities:**
1.1. Review Employer's Requirements (ER) **TBD** — specific ER sections for security systems
   - Identify CCTV coverage requirements (areas to be monitored, image quality requirements)
   - Identify access control requirements (controlled access points, access level structure)
   - Identify Terminal integration requirements per DEC-05
   - Identify performance criteria (resolution, frame rate, recording duration, etc.)

1.2. Review project objectives and constraints
   - OBJ-1: Safe & Reliable Operation — security supports safety monitoring
   - OBJ-5: Terminal Continuity — minimize disruption to existing operations
   - OBJ-6: Regulatory Compliance — applicable security and privacy regulations
   - OBJ-8: Future Expandability — design for Phase 2 growth

*Source: Decomposition Sections 2 and 7 (Objectives and Decisions)*

1.3. Conduct site survey **TBD** — coordinate with Employer for site access
   - Survey existing Terminal security infrastructure and integration points
   - Identify potential camera mounting locations and constraints
   - Verify existing conditions for cable routing and equipment installation
   - Photograph site conditions and potential mounting locations

1.4. Obtain base drawings from civil/structural/architectural disciplines
   - Site plan with property boundaries and key features
   - Building floor plans and elevations
   - Structural steel arrangement drawings
   - Underground utility corridor drawings

**Outputs:**
- Requirements summary matrix — ER requirements mapped to design response
- Site survey report with photographs
- Base drawing register

**Verification:** Requirements review meeting with project team and Employer (if applicable)

---

### Step 2: Design Development — Concept Design

**Objective:** Develop security system concept design addressing ER requirements.

**Activities:**
2.1. Develop CCTV coverage strategy
   - Define coverage zones based on risk assessment and ER requirements **TBD**
   - Identify coverage quality levels (identification, recognition, detection, monitoring)
   - Preliminary camera count and type selection (fixed vs. PTZ)

2.2. Develop access control strategy
   - Define access control zones and levels (public, controlled, restricted, high security)
   - Identify controlled doors/gates and required access control devices
   - Determine fail-safe vs. fail-secure requirements (coordinate with fire/life safety)

2.3. Develop system integration architecture
   - Define Terminal integration approach per DEC-05 (separate CCTV, access control interfaces)
   - Preliminary network topology and equipment room locations
   - Identify integration points and protocols

*Source: DEC-05 (Decomposition Section 7); Guidance.md — Design principles*

2.4. Prepare concept design sketches
   - Preliminary camera locations on site plan
   - Preliminary access control points on floor plans/site plan
   - Preliminary system architecture diagram

2.5. Interdisciplinary coordination — concept phase
   - Issue preliminary layouts to structural for mounting point feasibility
   - Coordinate with electrical for power supply strategy
   - Coordinate with lighting for illumination coverage
   - Coordinate with communications for network infrastructure

**Outputs:**
- Concept design sketches
- Preliminary equipment list and system architecture
- Concept design report or presentation for Employer review **TBD**

**Verification:**
- Concept design review meeting (internal and with Employer if applicable)
- Verify concept meets ER requirements
- Obtain feedback and direction for detailed design

---

### Step 3: Design Development — Detailed Design

**Objective:** Develop detailed security system design ready for drafting.

**Activities:**
3.1. Finalize camera locations and specifications
   - Perform camera coverage analysis using coverage calculation software
   - Verify coverage against ER requirements — eliminate blind spots in critical areas
   - Coordinate mounting heights and locations with structural disciplines
   - Select camera models and lens specifications **TBD** — based on coverage calculations

3.2. Finalize access control layout
   - Detail access control device locations (readers, controllers, locks, REX devices)
   - Develop door hardware schedule
   - Define access level/zone assignments
   - Coordinate with building design (PKG-21/22) for device mounting and power

3.3. Develop cable routing and power supply design
   - Route camera and access control cables from devices to equipment room(s)
   - Coordinate underground conduit routing with PKG-03
   - Coordinate overhead cable tray routing with PKG-17 and other disciplines
   - Verify power supply requirements and coordinate circuits with PKG-17

3.4. Develop network and integration design
   - Detail network topology (switches, firewalls, VLANs)
   - Define integration interface with Terminal security network
   - Specify NVR and VMS equipment locations
   - Specify access control panel and server locations

3.5. Develop typical details and notes
   - Camera mounting details (wall, pole, structural steel)
   - Conduit and cable entry details (especially for hazardous areas)
   - Access control device installation details
   - General notes and specifications for installation

3.6. Interdisciplinary coordination — detailed design phase
   - Issue coordination drawings to all affected disciplines
   - Attend IDC meetings and log/track comments
   - Resolve conflicts and update design accordingly
   - Obtain coordination sign-offs from all disciplines

**Outputs:**
- Finalized design calculations (camera coverage analysis)
- Equipment schedules and specifications
- Detailed cable routing plans
- Coordination comment log with resolutions

**Verification:**
- Coverage analysis review — verify all ER requirements met
- IDC comments all resolved and closed
- Design approved for drafting by discipline lead

---

### Step 4: Drafting and Drawing Production

**Objective:** Produce formal construction drawings per project CAD standards.

**Activities:**
4.1. Set up CAD files per project CAD standards **TBD**
   - Establish drawing numbering scheme
   - Set up drawing templates and title blocks
   - Import base drawings from civil/structural/architectural
   - Establish layers, line types, and symbology per CAD manual

4.2. Produce CCTV layout drawings
   - Site plan showing all camera locations with identifiers
   - Camera coverage zone overlays
   - Cable routing and conduit/cable tray paths
   - Equipment room locations (NVR, VMS, network equipment)
   - Legend and general notes

4.3. Produce camera coverage drawings
   - Field-of-view diagrams for each camera or coverage zone
   - Coverage analysis results (overlays showing coverage quality)
   - Notes on camera specifications, mounting height, lens selection
   - Coordination notes for lighting (reference PKG-18 drawings)

4.4. Produce access control layout drawings
   - Building floor plans or site plan showing all access control devices
   - Door hardware schedule and device identifiers
   - Access control panel/controller locations
   - Cable routing from devices to panels to central system
   - Access level zone designations
   - Legend and general notes

4.5. Produce system architecture and typical details (if required)
   - Network topology diagram
   - Integration interface diagram (Terminal connection)
   - Typical mounting and installation details
   - Conduit sealing details for hazardous areas **TBD**

4.6. Populate drawing register and cross-references
   - Maintain drawing list with all drawing numbers and titles
   - Ensure cross-references between drawings are correct and current
   - Verify all referenced drawings exist

**Outputs:**
- Complete drawing set in CAD format (DWG or native format)
- PDF versions for review and distribution

**Verification:**
- Drafter self-check — completeness, accuracy, CAD standards compliance
- Drawing register complete and accurate

### Step 5: Self-Check and Quality Control

**Objective:** Originator verification of drawing quality and completeness.

**Activities:**
5.1. Self-check by originator/drafter
   - Verify all ER requirements addressed in drawing set
   - Check dimensional accuracy and coordination with base drawings
   - Verify CAD standards compliance (layers, line types, text styles, etc.)
   - Check drawing numbering and cross-references
   - Verify all coverage analysis results and calculations support the design
   - Check for completeness — no missing information or TBD items remaining **TBD**

5.2. Run automated CAD quality checks (if available) **TBD**
   - Layer naming and usage
   - Text style and size compliance
   - Line type and weight compliance
   - Block/symbol library compliance

5.3. Correct deficiencies found during self-check
   - Document changes in design revision log
   - Update drawings and verify corrections

**Outputs:**
- Self-checked drawing set
- Self-check sign-off (internal quality record)

**Verification:**
- Self-check sign-off by originator
- Drawings ready for interdisciplinary check

### Step 6: Interdisciplinary Check (IDC)

**Objective:** Coordinate with other disciplines and resolve interface conflicts.

**Activities:**
6.1. Issue drawings for IDC review
   - Distribute drawing set to all affected disciplines (civil, structural, electrical, communications, fire protection, buildings)
   - Issue IDC review request via project document management system **TBD**

6.2. Attend IDC review meetings
   - Present design and highlight interface points
   - Discuss conflicts or coordination issues
   - Document comments and agreed resolutions

6.3. Track IDC comments and resolutions
   - Log all IDC comments in coordination tracking log
   - Assign responsibility for resolution (security systems designer or other discipline)
   - Track status to closure

6.4. Incorporate IDC comment resolutions
   - Update drawings per agreed resolutions
   - Issue revised drawings for re-check if major changes
   - Obtain sign-off from commenting disciplines that issues are resolved

**Outputs:**
- IDC comment log with all comments closed
- Updated drawing set incorporating IDC resolutions
- IDC sign-offs from all disciplines

**Verification:**
- All IDC comments resolved and closed
- No outstanding interdisciplinary conflicts
- Sign-offs obtained from key interface disciplines

---

### Step 7: Independent Technical Check

**Objective:** Independent peer review by qualified checker.

**Activities:**
7.1. Assign independent checker
   - Checker shall not be involved in original design (independent review)
   - Checker shall be qualified security systems designer or P.Eng. **TBD**

7.2. Checker review activities
   - Verify design meets ER requirements and project objectives
   - Check calculations and coverage analysis for accuracy and completeness
   - Verify compliance with applicable codes and standards
   - Check drawing quality, clarity, and constructability
   - Verify interdisciplinary coordination completeness
   - Check for errors, omissions, or conflicts

7.3. Issue check comments
   - Document check comments in formal comment log
   - Classify comments by severity (major/minor) **TBD** — per project QC procedures
   - Return commented drawings to originator for resolution

7.4. Resolve check comments
   - Designer addresses all check comments
   - Update drawings and calculations as required
   - Provide written responses to checker explaining resolutions
   - Resubmit for re-check if major changes

7.5. Checker verification and sign-off
   - Checker verifies all comments satisfactorily resolved
   - Checker signs off drawings as checked and approved
   - Document check completion in project quality records

**Outputs:**
- Check comment log with all comments resolved
- Checked and signed drawings
- Checker sign-off record

**Verification:**
- All check comments resolved to checker's satisfaction
- Checker sign-off obtained
- Drawings ready for discipline lead approval
