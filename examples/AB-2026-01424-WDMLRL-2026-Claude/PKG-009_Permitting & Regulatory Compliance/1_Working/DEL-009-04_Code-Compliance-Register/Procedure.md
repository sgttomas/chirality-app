# Procedure
## DEL-009-04 | Code Compliance Register & Inspection Log

---

## Purpose

This procedure describes the steps to establish, populate, and maintain the Code Compliance Register & Inspection Log (DEL-009-04) for the West Dried Meat Lake Regional Landfill Maintenance Shop Addition project. It covers: (1) producing the register framework before construction begins; and (2) operating the register throughout construction and the guarantee period to meet the inspection coordination and reporting requirements of the RFP (§3.3.2) and the CCDC 14–2013 contract.

## Steps

### Phase 1 — Establish Register Framework (Before Construction Begins)

#### Step 1.1 — Design Register Schema
Design the register structure including the following fields (at minimum):

- Entry ID
- Permit type (Development / Building / Safety Code)
- Permit reference number
- Condition number
- Condition text (verbatim from permit)
- Inspection type required
- Pass/fail criterion (specific code section or inspection standard reference) *(Enrichment: A-003)*
- Responsible party
- Planned inspection date
- Authority / inspector
- County rep notification date
- County rep attendance confirmation
- Inspection date (actual)
- Inspection outcome (Pass / Conditional Pass / Deficiency)
- Deficiency description
- Corrective action required
- Corrective action assigned to
- Target resolution date
- Actual resolution date
- Inspection report reference
- Date report delivered to County
- Compliance evidence reference
- Evidence type (signed certificate / authority letter / signed report / photo with sign-off) *(Enrichment: F-002)*
- Register operator (person who created/modified the entry) *(Enrichment: E-004)*
- Owner instruction date (guarantee period deficiencies only) *(Enrichment: D-001)*
- 10-day deadline (calculated, guarantee period deficiencies only) *(Enrichment: D-001)*
- Status (OPEN / RESOLVED / CLOSED)
- Regulatory change flag (Y/N — indicates items added mid-lifecycle per REQ-012) *(Enrichment: C-002)*

**Output:** Approved register template / schema.

#### Step 1.2 — Establish Supporting Filing Structure
Create a compliance evidence filing structure to receive:
- Permit documents (from DEL-009-01, DEL-009-02, DEL-009-03)
- Inspection reports
- Deficiency notices
- Corrective action records
- Sign-off confirmations
- Register audit records *(Enrichment: A-004)*

ASSUMPTION: file naming and storage conventions should align with the project document management system used by the Proponent.

#### Step 1.3 — Confirm County Project Representative and Communication Protocol
Confirm the identity of the County project representative (to be provided at project award, per RFP §3.1) and establish the protocol for:
- Advance notice of inspections (ASSUMPTION: minimum 48-hour notice — specific notice period TBD by County)
- Method of inspection report delivery to County
- Timeline for inspection report delivery to County (TBD — see Guidance Conflict Table OQ-005) *(Enrichment: A-002)*

**Output:** Documented communication protocol; contact information recorded.

### Phase 2 — Populate Register with Permit Conditions (Upon Permit Issuance)

#### Step 2.1 — Load Development Permit Conditions
Upon receipt of DEL-009-01 (Development Permit approval and conditions):
1. Review all conditions attached to the development permit.
2. Create a register entry for each condition that requires an inspection, sign-off, or ongoing compliance action.
3. Identify the relevant inspection authority and timing for each condition.
4. Record permit reference number and condition text verbatim.
5. Define the pass/fail criterion for each condition entry. *(Enrichment: A-003)*

#### Step 2.2 — Load Building Permit Conditions
Upon receipt of DEL-009-02 (Building Permit approval and conditions):
1. Review all conditions attached to the building permit.
2. Create a register entry for each condition requiring inspection, sign-off, or ongoing compliance action.
3. Identify the relevant inspection authority (Camrose County building inspection) and required inspection stage for each condition.
4. Record permit reference number and condition text verbatim.
5. Define the pass/fail criterion for each condition entry. *(Enrichment: A-003)*

Common building permit inspection stages to expect (ASSUMPTION — typical for Alberta building construction; confirm with authority upon permit issuance):
- Foundation / footing inspection
- Framing / structural inspection
- Insulation and vapour barrier inspection
- Mechanical / HVAC rough-in inspection
- Electrical rough-in inspection (if under building permit scope)
- Plumbing rough-in inspection (if under building permit scope)
- Final building inspection

#### Step 2.3 — Load Safety Code Permit Requirements
Upon receipt of DEL-009-03 (Safety Code Permits):
1. Review all Safety Code permits issued and their associated conditions and inspection requirements.
2. Create register entries for each Safety Code inspection required.
3. Identify the relevant Safety Codes authority for each permit type.
4. Define the pass/fail criterion for each Safety Code inspection entry. *(Enrichment: A-003)*
5. Note that Safety Code inspections may be required at multiple construction stages and may involve multiple technical disciplines. (ASSUMPTION — specific Safety Code disciplines TBD pending permit issuance.)

#### Step 2.4 — Build Inspection Schedule Matrix
Using the populated register conditions and the project construction schedule:
1. Map each required inspection to the corresponding construction milestone or stage.
2. Sequence inspections to avoid blocking critical path activities.
3. Identify inspections that require sequential sign-off (i.e., later work cannot proceed until prior inspection is passed).
4. Flag inspections requiring longest lead time for authority scheduling.

**Output:** Inspection schedule matrix linked to project schedule.

#### Step 2.5 — Code Identification Completeness Check *(New — Enrichment: C-001)*
After all permit conditions have been loaded (Steps 2.1–2.3):
1. Review the loaded conditions against the list of applicable codes and standards (see Specification Standards table).
2. Verify that the upstream permits (DEL-009-01, DEL-009-02, DEL-009-03) have addressed all known applicable codes for the project scope.
3. If gaps are identified (codes not addressed by any permit), flag them and consult with the relevant regulatory authority.
4. Record the baseline condition count and baseline inspection count in the Datasheet (Register Content Attributes) per X-004.

ASSUMPTION: This check may require consultation with regulatory authorities. If gaps cannot be resolved before construction begins, add them to the register as open items with a "TBD" status and escalate to the Project Manager.

### Phase 3 — Operate Register During Construction (Ongoing)

#### Step 3.1 — Monitor Construction Milestones Against Inspection Schedule
On a rolling basis:
1. Review upcoming inspection trigger points against the construction schedule.
2. Initiate inspection requests sufficiently in advance of the required inspection date to allow for authority scheduling.
3. Record inspection request submission date in the register.

#### Step 3.2 — Notify County Project Representative
For each upcoming inspection:
1. Notify the County project representative in advance (minimum notice period per confirmed protocol — TBD).
2. Record notification date in the register.
3. Confirm County representative attendance before the inspection proceeds.

Note: RFP §3.3.2 requires the County project representative to be present at all inspections. Do not proceed with an inspection without confirming County attendance arrangements.

#### Step 3.3 — Conduct Inspection and Record Outcome
On the day of inspection:
1. Facilitate the inspection with the regulatory authority.
2. Ensure County project representative is present.
3. Obtain the inspection report or outcome notice from the authority.
4. Record the following in the register:
   - Actual inspection date
   - Name / authority of inspector
   - County rep attendance (confirmed Y/N)
   - Inspection outcome (Pass / Conditional Pass / Deficiency)
   - Report reference number
   - Register operator name (person recording the entry) *(Enrichment: E-004)*

#### Step 3.4 — Deliver Inspection Report to County
Within [TBD — confirm with County; see Guidance Conflict Table OQ-005] business days of inspection completion:
1. Provide a copy of the completed inspection report to the County project representative.
2. Record the delivery date and method in the register.

Source: RFP §3.3.2 — "shall be provided with a copy of all complete inspection reports."

#### Step 3.5 — Log and Manage Deficiencies
If an inspection results in a deficiency:
1. Create a deficiency entry in the register with:
   - Deficiency description (verbatim from inspection report)
   - Responsible party for corrective action
   - Target resolution date
2. Assign the corrective action to the responsible subcontractor or trade.
3. Monitor progress against the target resolution date.
4. When corrective action is complete, arrange re-inspection if required by the authority.
5. Upon re-inspection pass or authority sign-off, update the register status to RESOLVED.

Note: During the guarantee period, deficiency rectification must be completed within 10 days of the Owner's written instruction (RFP §2.10.6). Record the Owner's instruction date and the calculated 10-day deadline in the register. See Guidance / Operationalizing the 10-Day Deficiency Rectification Obligation for rationale. *(Enrichment: D-001)*

#### Step 3.6 — Incorporate Mid-Lifecycle Regulatory Changes *(New — Enrichment: C-002)*
If regulatory changes or newly discovered obligations arise during construction:
1. Identify the source of the new obligation (code amendment, additional authority requirement, new permit condition).
2. Create new register entries for each new condition or inspection requirement. Set the "Regulatory change flag" to Y.
3. Update the inspection schedule matrix to incorporate new inspection requirements.
4. Record the date and source of the new obligation in the register.
5. Notify the Project Manager and update the baseline counts in the Datasheet if applicable.

#### Step 3.7 — Periodic Register Review *(New — Enrichment: A-005, A-004, D-003)*
At the defined audit frequency (TBD — see REQ-011; ASSUMPTION: monthly during active construction or at each major construction milestone):
1. Review the register for completeness: verify all triggered inspections have been requested and tracked.
2. Verify all deficiencies have been logged and assigned with target resolution dates.
3. Verify all inspection reports have been delivered to County.
4. Assess register currency: confirm no construction milestones have passed without the corresponding inspection being recorded.
5. Calculate and record KPI metrics (REQ-014): inspection request lead time, County notification compliance rate, inspection report delivery time, deficiency resolution time, register completeness ratio.
6. Prepare compliance status summary report (see Guidance / Compliance Status Reporting) and distribute to stakeholders.
7. Document the review findings and any corrective actions in the register audit record.

**Output:** Register audit record; KPI summary report; compliance status summary.

---

### Phase 4 — Maintain Register Through Guarantee Period

#### Step 4.1 — Maintain Active Register Post-CCC
The register remains active for the full guarantee period (construction period plus 2 years post-CCC — RFP §2.10.2). Do not archive or close the register at practical completion.

#### Step 4.2 — Log Guarantee Period Deficiencies
Deficiencies discovered during the guarantee period (by Owner inspection or observation — RFP §2.10.4) shall be logged using the same deficiency tracking process as Phase 3, Step 3.5. Record the Owner's written instruction date and calculate the 10-day deadline (RFP §2.10.6). The 10-day clock starts at the date of the Owner's written instruction. See Guidance / Operationalizing the 10-Day Deficiency Rectification Obligation for detailed rationale. *(Enrichment: D-001)*

---

### Phase 5 — Close Out Register at Project Completion

#### Step 5.1 — Verify All Conditions Closed
Before CCC application:
1. Review all register entries.
2. Confirm status of every condition entry is CLOSED (inspection passed; no outstanding deficiencies).
3. Identify any open entries and resolve or escalate.

#### Step 5.2 — Compile Final Compliance Summary
Prepare a Final Compliance Summary document that:
- Lists all permits tracked
- Lists all inspections completed with outcomes
- Confirms all conditions are met
- References the compliance evidence file
- Summarizes KPI performance across the construction phase *(Enrichment: F-004)*
- Includes count of mid-lifecycle regulatory changes incorporated (if any) *(Enrichment: C-002)*

This summary is a supporting document for the CCC package (DEL-020-02).

#### Step 5.3 — Deliver Final Compliance Package to County
Deliver the completed register and compliance evidence file to the County as part of project closeout documentation.
